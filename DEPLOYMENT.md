# Deployment Process

## Staging Deployment

Staging deployment happens after approved changes are merged into the `staging` branch.

The staging environment is used to test changes before production.

## Production Deployment

Production deployment happens after tested changes are merged into the `main` branch.

Only approved and tested code should be released to production.

## Deployment Notes

Each production deployment should include:

- Deployment date
- Deployed branch
- Version or tag
- Summary of changes
- Deployment status
